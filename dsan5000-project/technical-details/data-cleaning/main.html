Data Cleaning and Preprocessing

1. Introduction and Motivation

Data cleaning is a critical step to ensure the quality and reliability of our analysis. The primary objective of this notebook is to prepare a robust dataset for supervised learning models to predict the Gentrification Index. By merging American Community Survey (ACS) 2019 data with our calculated Gentrification Index, we construct a feature-rich dataset capturing demographic, economic, and housing characteristics.

Our raw census data, while comprehensive, contained missing values and potential outliers that could skew our results. Additionally, we needed to merge our socio-economic data with the target variable, the Gentrification Index.

This report documents our pipeline for:

  1. Merging datasets to create a unified view.

  2. Imputing missing values to preserve data integrity.

  3. Detecting and removing outliers to ensure robust modeling.

  4. Feature Selection to identify the most predictive variables.

Analysis of Missing Values

The initial check reveals that our dataset is remarkably clean, with the exception of 4 missing values in the gent_index_0_100 target variable. While standard practice often involves dropping rows with missing targets, we utilize MICE (Multiple Imputation by Chained Equations) here. This allows us to impute these few missing values based on the strong multivariate relationships present in the socio-economic features, preserving the integrity of the dataset without discarding information.

Handling Missing Data (Imputation)

Instead of dropping rows and losing valuable data, we employed Multiple Imputation by Chained Equations (MICE) via IterativeImputer. This advanced technique models each feature with missing values as a function of others, providing a more accurate estimation than simple mean/median imputation for correlated variables.

Code 
# Select numeric columns for imputation

numeric_cols = merged_df.select_dtypes(include=[np.number]).columns
cols_to_impute = [col for col in numeric_cols if col not in ['GEOID', 'year','tract','county','state','NAME']]

imputer = IterativeImputer(random_state=42)
merged_df[cols_to_impute] = imputer.fit_transform(merged_df[cols_to_impute])

# Verify no missing values
print("=" * 50)
print(f"Missing values after imputation: {merged_df[cols_to_impute].isnull().sum().sum()}")

3. Outlier Detection and Treatment

Method

To ensure our models are robust and not driven by extreme anomalies, we used Isolation Forest, an unsupervised anomaly detection algorithm. We set the contamination parameter to 0.05, estimating that approximately 5% of our census tracts might represent statistical outliers (extreme wealth, poverty, or population shifts not representative of general trends).

Code 
# Select numeric columns for outlier detection (already imputed)
numeric_cols_clean = merged_df.select_dtypes(include=[np.number]).columns

# Drop identifiers from numeric cols
numeric_cols_clean = numeric_cols_clean.drop(['GEOID', 'year','tract','county','state','NAME'], errors='ignore')

clf = IsolationForest(random_state=42, contamination=0.05)
outliers = clf.fit_predict(merged_df[numeric_cols_clean])

# Identify outliers
outlier_data = merged_df[outliers == -1]
print(f"Number of outliers removed: {len(outlier_data)}")


cleaned_df = merged_df[outliers == 1].copy()

Analysis of Detected Outliers

The Isolation Forest algorithm identified approximately 3,700 outliers (5% of data). A preliminary inspection of the removed rows (shown above) suggests these tracts often exhibit:

  • Extreme Population Counts: Very low or high counts in specific demographic categories (e.g., 0 Asian population in some tracts).
  • Unusual Housing Dynamics: High vacancy rates or extremely high/low median property values.

By removing these anomalies, we ensure that our supervised learning models focus on generalizable patterns rather than fitting to edge cases.

4. Data Type Correction and Formatting

Before processing, we ensured all identifier columns (GEOID, state, county) were treated as strings or appropriate codes, while socio-economic metrics were converted to numeric types. This was handled during the initial data load and merge phase.

5. Normalization and Scaling

Decision: Although StandardScaler was imported in our environment, we decided to perform normalization (scaling features to mean=0, variance=1) during the Modeling phase (e.g., inside a Scikit-Learn Pipeline). This prevents data leakage, where information from the test set could inadvertently influence the scaling parameters.

6. Subsetting the Data

We filtered the dataset to exclude rows identified as outliers. This subsetting ensures that our downstream EDA and Supervised Learning models focus on the “core” distribution of US census tracts.

  • Original Rows: ~74,000
  • Filtered Rows: ~70,300

Interpretation of Selected Features

The intersection of Filter, Lasso, and Random Forest methods highlights a core set of 11 features that are robust predictors of gentrification. The selected features cluster around three main themes. These features align with sociological theories of gentrification, validating the data-driven selection process.

  1. Economic Status: Income below poverty level, Household Income $200,000 or more, Households receiving food stamps/SNAP.

  2. Demographic: Black or African American alone, Foreign born, Male/Female Management, business, science, and arts occupations - indicating professional class presence.

  3. Occupation & Education: Male/Female Management, business, science, and arts occupations - indicating professional class presence.

  4. Gender: Male/Female Management, business, science, and arts occupations - indicating professional class presence.

  5. Commuting & Housing: Public transportation, Commute by bicycle, Owner-occupied housing units - proxies for urban density and lifestyle associated with gentrifying areas.

Code 
# Define target variable
target = "gent_index_0_100"


# Verify columns exist
# First, drop non-numeric columns for feature selection except target if needed
numeric_df = cleaned_df.select_dtypes(include=[np.number])
# Drop identifiers
numeric_df = numeric_df.drop(columns=['GEOID', 'year', 'tract', 'county', 'state'], errors='ignore')


# Drop missing values before feature selection
numeric_df = numeric_df.dropna()


# Prepare for Feature Selection
X_fs = numeric_df.drop(columns=[target], errors='ignore')
y_fs = numeric_df[target]

# 1. Filter Method: SelectKBest
selector = SelectKBest(score_func=f_regression, k=10)
X_new = selector.fit_transform(X_fs, y_fs)
selected_features_filter = X_fs.columns[selector.get_support()]
print(f"Selected features (Filter Method): {selected_features_filter.tolist()}")

# 2. Embedded Method: Lasso
scaler = StandardScaler()
X_scaled_fs = scaler.fit_transform(X_fs)
lasso = LassoCV(cv=5, random_state=42, max_iter=100000).fit(X_scaled_fs, y_fs)
model = SelectFromModel(lasso, prefit=True)
selected_features_lasso = X_fs.columns[model.get_support()]
print(f"Selected features (Lasso): {selected_features_lasso.tolist()}")

# 3. Embedded Method: Random Forest
rf = RandomForestRegressor(n_estimators=100, random_state=42)
rf.fit(X_fs, y_fs)
importances = pd.Series(rf.feature_importances_, index=X_fs.columns)
selected_features_rf = importances.nlargest(10).index
print(f"Selected features (Random Forest): {selected_features_rf.tolist()}")


# --- Voting Mechanism ---
# Combine all selected features
all_selected = (selected_features_filter.tolist() + 
                selected_features_lasso.tolist() + 
                selected_features_rf.tolist())

# Count occurrences of each feature
from collections import Counter
feature_counts = Counter(all_selected)

# Select features that were chosen by at least 2 methods
selected_features = [feat for feat, count in feature_counts.items() if count >= 2]

print(f"\nOverlapped features (>=2 methods): {len(selected_features)}")
print(f"Features: {selected_features}")

# Save selected features to CSV
pd.DataFrame(selected_features, columns=['feature']).to_csv("../../data/processed-data/selected_features.csv", index=False)
print("\nSaved selected features to '../../data/processed-data/selected_features.csv'")
